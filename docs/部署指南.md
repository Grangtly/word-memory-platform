# 部署指南

## 实际方案（已上线）

| 层 | 平台 | 说明 |
|---|------|------|
| 前后端一体 | **Railway** | 同一服务托管前端 + 后端，无需 Vercel |
| 代码托管 | **GitHub** | Grangtly/word-memory-platform |

**部署地址**：`https://word-memory-platform-production.up.railway.app/login`（一个地址同时有页面和 API，已上线）

---

## 前置条件

1. 注册 [GitHub](https://github.com) 账号
2. 注册 [Railway](https://railway.com) 账号（GitHub 一键登录，免费额度）

> 不需要绑卡、不需要手机验证、不需要 Vercel。

---

## 步骤 1：推送代码到 GitHub

```bash
git remote add origin https://github.com/你的用户名/word-memory-platform.git
git branch -M main
git push -u origin main
```

---

## 步骤 2：Railway 部署

1. 打开 [Railway](https://railway.app/dashboard)，GitHub 登录
2. **New Project** → **Deploy from GitHub** → 选择仓库
3. 项目根目录需要有这些文件（已配好）：
   - `Dockerfile`（多阶段构建：先编客户端再编服务端）
   - `railway.toml`（指定 builder = "DOCKERFILE"）
4. Railway 自动构建 → 成功后顶部显示 **Public URL**

---

## 步骤 3：验证

浏览器打开 Railway 地址 → 注册 → 登录 → 背单词

---

## 构建原理

```
Dockerfile:
  阶段1: 编译 Vue 前端 (client/)
          npm install → npm run build → dist/

  阶段2: 构建后端 (server/)
          npm install
          复制 阶段1 的 dist/ → server/public/

  运行时:
          Express 同时提供:
          - /api/*      → 后端接口
          - /*          → 前端页面 (SPA)
```

---

## 踩过的坑

| 坑 | 原因 | 解决 |
|----|------|------|
| nixpacks `$NIXPACKS_PATH` 报错 | Railway 平台 bug | 改用 Dockerfile 构建 |
| Express 5 `app.get('*')` 崩溃 | path-to-regexp v8 不支持 `*` | 改用正则 `/^\/(?!api\/).*/` |
| `railway.toml` builder 被拒 | `"docker"` 必须大写 | 改为 `"DOCKERFILE"` |

---

## 步骤 4：UptimeRobot 保活（防止休眠）

Railway 免费实例 15 分钟无请求会自动休眠，下次访问等待 30-60 秒。

1. 打开 [UptimeRobot](https://uptimerobot.com)，注册免费账号
2. **+ Create Monitor** → 选 **HTTP(s)**
3. URL 填 `https://word-memory-platform-production.up.railway.app/login`
4. 间隔选 **5 minutes**
5. 点 **Create**

> 之后 UptimeRobot 每 5 分钟自动 ping 一次，网站永不休眠。
