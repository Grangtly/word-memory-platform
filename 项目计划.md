## 第一部分：JSON 文件版后端开发

### 步骤 1：初始化项目
```bash
mkdir word-memory-platform
cd word-memory-platform
mkdir server
cd server
npm init -y
npm install express cors jsonwebtoken bcryptjs
```

### 步骤 2：创建目录结构
```
server/
├── index.js
├── routes/
│   ├── auth.js
│   └── words.js
├── data/
│   ├── words.json      # 单词库
│   ├── users.json      # 初始写 []
│   └── records.json    # 初始写 []
└── utils/
    └── schedule.js     # 遗忘曲线算法
```

### 步骤 3：编写入口文件 `index.js`
- 创建 Express 应用
- 配置 `cors()` 和 `express.json()`
- 挂载认证路由 `/api/auth` 和单词路由 `/api/words`
- 监听 3000 端口

### 步骤 4：编写单词库 `data/words.json`
- JSON 数组，每个对象包含 `id, word, phonetic, meaning, example`
- 初期放 10-20 个单词跑通流程

### 步骤 5：编写工具函数
- `readJSON(filePath)` —— 读取 JSON 文件
- `writeJSON(filePath, data)` —— 写入 JSON 文件

### 步骤 6：编写认证路由 `routes/auth.js`
- `POST /api/auth/register` —— 接收 username/password，bcrypt 加密，存入 users.json，返回成功
- `POST /api/auth/login` —— 验证密码，生成 JWT Token，返回 token 和 username
- 编写 `authMiddleware` 中间件 —— 解析 Token，挂载 `req.userId`

### 步骤 7：编写遗忘曲线算法 `utils/schedule.js`
- 定义复习间隔数组 `[5分钟, 30分钟, 12小时, 1天, 2天, 4天, 7天, 15天, 30天]`
- 导出 `calculateNextReview(stage, remembered)`
  - 记住了 → stage+1，`nextReviewAt = now + intervals[nextStage]`
  - 没记住 → stage=0，`nextReviewAt = now + 5分钟`

### 步骤 8：编写单词路由 `routes/words.js`
- `GET /api/words/today` —— 查该用户待复习单词，有则返回，无则返回 10 个新单词
- `POST /api/words/:id/review` —— 接收 `{ remembered: bool }`，调用算法更新/创建记录

### 步骤 9：后端自测
- 用 Postman 测试各接口
- 确认注册、登录、获取单词、提交复习都能正常返回数据

## 第二部分：Vue 3 前端开发

### 步骤 1：初始化项目
```bash
# 回到项目根目录
cd ..
npm create vite@latest client -- --template vue
cd client
npm install
npm install vue-router axios pinia
```

### 步骤 2：配置代理
在 `vite.config.js` 中添加：
```javascript
server: {
  proxy: {
    '/api': 'http://localhost:3000'
  }
}
```

### 步骤 3：创建 API 封装 `src/api.js`
- 创建 Axios 实例，baseURL 为 `/api`
- 请求拦截器：从 Pinia/localStorage 取 Token 加入 Authorization 头
- 导出：`register()`, `login()`, `getTodayWords()`, `submitReview()`

### 步骤 4：创建 Pinia Store `src/stores/user.js`
- state：`token`, `username`
- actions：`login()` 调接口并保存 token，`logout()` 清除状态
- 持久化：token 同步存 localStorage

### 步骤 5：创建路由 `src/router/index.js`
- `/login` → Login 页
- `/home` → Home 页（需登录）
- `/learn` → Learn 页（需登录）
- 路由守卫 `beforeEach`：没 token 时跳 `/login`

### 步骤 6：创建登录页 `src/views/Login.vue`
- 表单：用户名输入框 + 密码输入框
- 登录按钮：调 `login()` 接口，成功后跳 `/home`
- 注册按钮：调 `register()` 接口，成功后提示“注册成功请登录”

### 步骤 7：创建单词学习页 `src/views/Learn.vue`
- 页面进入时调 `getTodayWords()`，拿到单词列表
- 单词卡片组件：显示英文单词、音标、释义、例句
- 两个按钮：
  - “没记住”（红色）→ 调 `submitReview(id, false)`，切下一个
  - “记住了”（绿色）→ 调 `submitReview(id, true)`，切下一个
- 单词全部学完显示“今日任务完成”

### 步骤 8：创建首页 `src/views/Home.vue`
- 显示学习统计（可调用一个统计接口，或从 Learn 页传参）
- “开始学习”按钮 → 跳转 `/learn`
- “退出登录”按钮 → 清除 token → 跳 `/login`

### 步骤 9：修改 `App.vue`
- 放 `<router-view />` 即可

## 第三部分：前后端联调

### 步骤 1：同时启动
```bash
# 终端1：启动后端
cd server
node index.js

# 终端2：启动前端
cd client
npm run dev
```

### 步骤 2：走完整流程
1. 浏览器打开 `http://localhost:5173`
2. 注册一个账号
3. 登录
4. 进入学习页，背单词
5. 点击“记住了”/“没记住”
6. 观察后端 `data/records.json` 是否生成记录
7. 明天再登录（或手动改 `nextReviewAt` 为过去时间），验证复习功能

### 步骤 3：修复问题
- 跨域 → 检查代理配置
- Token 过期 → 检查 JWT 有效期
- 数据格式不对 → 对照 API 返回格式调整前端

## 第四部分：部署上线

### 步骤 1：部署后端
- 推荐平台：Railway / Render（免费额度）
- 把 `server/` 文件夹推送到 GitHub
- 在平台上导入仓库，设置启动命令 `node index.js`
- 获得后端公开地址，如 `https://xxx.onrender.com`

### 步骤 2：部署前端
- 推荐平台：Vercel / Netlify
- 修改 `src/api.js` 的 baseURL 为后端公开地址
- `npm run build` 生成 `dist/`
- 把 `client/` 推送到 GitHub
- Vercel 导入仓库，自动构建部署
- 获得前端公开地址，如 `https://xxx.vercel.app`

### 步骤 3：绑定域名（可选）
- 在域名商添加 CNAME 记录指向 Vercel 地址
- 在 Vercel 设置中添加自定义域名

## 第五部分：升级到 MongoDB

### 步骤 1：安装 MongoDB 和 Mongoose
```bash
cd server
npm install mongoose
```

### 步骤 2：创建数据库连接 `server/db.js`
- 连接本地 MongoDB 或 Atlas 云端数据库

### 步骤 3：创建数据模型 `server/models/`
- `User.js` —— username, password
- `Word.js` —— word, phonetic, meaning, example
- `Record.js` —— userId, wordId, stage, nextReviewAt, reviewCount

### 步骤 4：改造路由
- `auth.js`：`readJSON/writeJSON` → `User.findOne()` / `User.create()`
- `words.js`：手动过滤 → `Record.find().populate('wordId')` / `record.save()`

### 步骤 5：导入单词库
- 写 `seed.js` 脚本，读取 `words.json` 批量写入 MongoDB，运行一次即可

### 步骤 6：删除旧文件
- 移除 `data/users.json`、`data/records.json`
- 移除 `readJSON`、`writeJSON` 工具函数

### 步骤 7：重新部署
- 推送更新后的代码
- 在部署平台设置 MongoDB 连接字符串环境变量
