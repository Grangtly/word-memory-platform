# 部署指南

## 推荐方案

| 层 | 平台 | 费用 | 特点 |
|---|------|------|------|
| 后端 | **Wispbyte** | 免费 | 24/7 不休眠，无需绑卡 |
| 前端 | **Vercel** | 免费 | Vite 原生支持，无需绑卡 |

---

## 前置条件

1. 注册 [GitHub](https://github.com) 账号
2. 注册 [Wispbyte](https://wispbyte.com) 账号（邮箱 + 密码即可，无需绑卡）
3. 注册 [Vercel](https://vercel.com) 账号（GitHub 一键登录，无需绑卡）

---

## 步骤 1：推送代码到 GitHub

```bash
# 在项目根目录执行
git remote add origin https://github.com/你的用户名/word-memory-platform.git
git branch -M main
git push -u origin main
```

---

## 步骤 2：Wispbyte 部署后端

1. 打开 [Wispbyte](https://wispbyte.com)，登录
2. 创建新服务器，选择 **Node.js**
3. 连接 GitHub 仓库，选择 `server` 目录
4. 设置启动命令：`node index.js`
5. 部署，获得后端地址，例如 `https://word-memory-api.wispbyte.com`

> Wispbyte 最大优势：**24/7 不休眠**。其他免费平台 5-15 分钟无人访问就自动休眠，下次访问要等 30-60 秒唤醒。

---

## 步骤 3：Vercel 部署前端

1. 打开 [Vercel](https://vercel.com)，用 GitHub 登录
2. 点击 **Add New** → **Project**
3. 选择你的 GitHub 仓库
4. 配置：
   | 选项 | 值 |
   |------|----|
   | Root Directory | `client` |
   | Framework Preset | Vite（自动识别） |
   | Build Command | `npm run build` |
   | Output Directory | `dist` |
5. 添加环境变量：
   - **Key**: `VITE_API_URL`
   - **Value**: `你的 Wispbyte 地址/api`（如 `https://word-memory-api.wispbyte.com/api`）
6. 点击 **Deploy**
7. 部署完成后获得前端地址，例如 `https://word-memory.vercel.app`

---

## 步骤 4：验证部署

1. 浏览器打开 Vercel 给你的地址
2. 注册 → 登录 → 开始学习 → 点击记住/没记住
3. 确认一切正常

---

## 重要提醒：数据持久化

Wispbyte 容器重新部署时，容器内的 `users.json` 和 `records.json` **会被重置**。

- **短期影响**：使用期间不做重新部署则不受影响
- **根本解决**：完成第五阶段 MongoDB 升级后，数据存入 MongoDB Atlas 云端数据库，与 Wispbyte 容器分离，永久保存

## 可选：绑定自定义域名

### 前端 (Vercel)
1. Vercel 项目 → Settings → Domains → 添加域名
2. 域名商添加 CNAME 记录，值指向 `cname.vercel-dns.com`

### 后端 (Wispbyte 或其他)
1. 部署平台项目设置 → Domains → 添加域名
2. 域名商添加对应 CNAME 记录
