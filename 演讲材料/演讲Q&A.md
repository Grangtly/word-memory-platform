# 演讲 Q&A 问答准备

## 关于原理

### Q1：艾宾浩斯遗忘曲线是什么？靠谱吗？

这是德国心理学家赫尔曼·艾宾浩斯在 1885 年通过实验发现的记忆规律：人在学习后遗忘速度先快后慢。基于这个规律，在遗忘临界点复习，记忆效率最高。一百多年来被教育领域广泛验证，是现代间隔重复学习法（Spaced Repetition）的理论基础。

### Q2：你们的算法具体怎么运作？

系统预设了 9 个复习间隔：5分钟 → 30分钟 → 12小时 → 1天 → 2天 → 4天 → 7天 → 15天 → 30天。当你点击「记住了」，单词的复习间隔晋级（比如从1天到2天）；点击「没记住」，级别归零，5分钟后重新学。每个单词独立追踪，实现了自适应的个性化复习。

### Q3：和市面上的背单词软件有什么不同？

优势：免费、开源、数据你自己可控。不需要下载 App，浏览器打开就能用。200 个四六级核心词精心整理。最重要的是——代码完全透明，任何人都可以查看和二次开发。

## 关于功能

### Q4：系统有哪些核心功能？

五个核心功能：
- 智能学习：自动推送今天该复习的单词
- 学习统计：已学单词数、掌握率、今日复习数
- 每日打卡：记录连续学习天数
- 单词列表：200 词完整列表，标注学习状态
- 测验模式：4 选 1 选择题，自动评分

### Q5：目前有多少单词？能加更多吗？

当前 200 个四六级核心高频词。系统支持增量扩展，随时可以加入考研、雅思、托福等更多词库。添加方式只需更新一个 JSON 文件，系统自动识别新词。

### Q6：能发音吗？

当前版本暂未集成发音功能。这是下一步短期待完成的功能之一——使用浏览器的 TTS（文字转语音）引擎，无需任何外部依赖即可朗读单词。

## 关于技术

### Q7：系统是怎么搭建的？用了什么技术？

前端 Vue.js（网页界面框架）+ 后端 Node.js（服务端运行环境）+ MongoDB Atlas（云端数据库）。全部部署在 Railway 免费云平台上，通过 Dockerfile 统一构建。用一个网址就能访问，不需要安装 App。

### Q8：花了多少钱？

0 元。所有的技术组件都有免费套餐：
- Railway：免费云服务器（512MB 内存）
- MongoDB Atlas：免费云数据库（512MB 存储）
- UptimeRobot：免费保活监控
- GitHub：免费代码托管

### Q9：怎么保持服务不中断？

Railway 免费套餐的服务器在无人访问 15 分钟后会自动休眠。我们用了 UptimeRobot 保活服务，每 5 分钟自动访问一次网站，确保服务器始终保持活跃状态。完全免费。

## 关于数据与隐私

### Q10：用户的数据安全吗？会不会丢失？

密码使用 bcrypt 加密存储（不可逆哈希），即使数据库泄露也无法还原密码。所有用户的学习记录通过账号隔离，互不可见。数据存在 MongoDB Atlas 云端，与服务器容器分离——服务器重启或崩溃不会丢失数据。

### Q11：可以有多个用户吗？需要每个人注册吗？

可以。每个用户独立注册账号，登录后各自的学习记录完全隔离。目前免费套餐下可以支持约 50 人同时在线使用。

## 关于未来

### Q12：后续有什么改进计划？

短期（1 周内）：发音功能、夜间模式、四六级分类
中期（1 个月）：自建词库、词典 API 自动释义、学习图表
长期：考研/雅思词库、班级模式、AI 推荐算法、移动端 App

### Q13：能不能应用到其他学科？

完全可以。这个遗忘曲线算法的核心是通用的——只要有"需要记忆的内容"（单词、公式、历史事件、法律条文等），系统框架可以直接复用。后端 API 和间隔算法不需要改动，只需替换词库内容和前端界面即可。

### Q14：项目开源吗？别人能用吗？

是的，代码托管在 GitHub（Grangtly/word-memory-platform），完全公开。任何人都可以：
- 直接使用（打开网址注册即可）
- Fork 代码自己部署一套
- 基于现有框架二次开发
